The Traditional Web

Click a Link

Browser  →  Server
        ←  HTML Page
  • User clicks a link
  • Browser requests a full page
  • Server returns HTML
  • Browser displays the new page

The web has worked this way for 30 years.

Slide 2 of 8

The Problem

Small Change, Big Refresh

Want to:

  • Add an item to a cart
  • Save a form
  • Update a table row
But the browser reloads the entire page.

We often need less than a full page refresh.
Slide 3 of 8

Modern Solution

Single Page Applications

Browser  →  Server
        ←  JSON

JavaScript updates page
  • Request data as JSON
  • Receive JSON
  • JavaScript updates the UI

Powerful — but often complex.

Slide 4 of 8

HTMX

A Different Approach

Browser  →  Server
        ←  HTML Fragment
  • Request HTML
  • Receive HTML
  • Update part of the page

No JSON required.

Slide 5 of 8

The HTMX Idea

Extend HTML

<button hx-get="/hello">
  Click Me
</button>

HTMX adds attributes like:

hx-get hx-post hx-target hx-swap hx-trigger

HTML becomes interactive.

Slide 6 of 8

Three Things HTMX Needs

Every HTMX Request

What?
hx-get="/hello"
Where?
hx-target="#result"
How?
hx-swap="innerHTML"
Slide 7 of 8

Complete Example

Page HTML
<button
  hx-get="/hello"
  hx-target="#result">
  Say Hello
</button>

<div id="result"></div>
→
Server returns
<p>Hello World!</p>

Only the target updates. No page reload.

HTMX in One Sentence

The Mental Model

HTMX lets HTML make HTTP requests and update parts of a page with HTML returned from the server.

Everything else is built on that idea.

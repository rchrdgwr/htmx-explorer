15. Event System

Components talking to each other without being coupled together. htmx fires rich DOM events at every step — listen to them, or fire your own.

htmx lifecycle events
EventWhen
htmx:beforeRequestBefore XHR fires — can cancel

Decoupled components: The save form doesn't know about the grid, and the grid doesn't know about the form. They communicate through events on a shared ancestor — clean and maintainable.

Key Points

1
Listen to htmx events to integrate third-party libraries (charts, maps, etc.)
2
hx-on:htmx:after-swap="..." runs inline JS after each swap
3
htmx.trigger(el, 'event') fires a custom event from JavaScript
4
Custom events decouple components — each piece only knows about its own job

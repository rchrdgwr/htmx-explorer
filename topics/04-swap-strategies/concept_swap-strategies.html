4. Swap Strategies

hx-swap

Once you have the response, hx-swap controls exactly how it is inserted into the DOM. Default is innerHTML.

innerHTML (default)
Replaces the contents of the target
<div id="t">Old</div> <div id="t">New</div>
outerHTML
Replaces the target element itself
<div id="t">Old</div> <section>New</section>
beforeend
Appends after existing content
Item 1 Item 2 + New item
afterbegin
Prepends before existing content
+ New item Item 1 Item 2
delete
Removes the target, ignores response body
Other content <div id="t">Gone</div>
none
Do not swap anything (useful for side effects)
Response is discarded - fire and forget
Chat / feed pattern: Use beforeend to append new messages, afterbegin to prepend newest-first. No page reload, no lost scroll position.

Key Points

1
innerHTML is the default - you only need to specify hx-swap when you want something different
2
outerHTML + hx-target="this" is the inline-edit pattern: replace the button with a form
3
beforeend is perfect for chat messages, activity feeds, and "load more" lists
4
Add swap:200ms and settle:100ms modifiers to animate transitions

3. Targeting

Where does the response go? hx-target accepts any CSS selector — most of htmx's power comes from getting targeting right.

hx-target
SelectorWhat it targets
#contentElement with id="content"
thisThe element that triggered the request
closest trNearest ancestor <tr> — great for table rows
closest .cardNearest ancestor with class "card"
next .resultNext sibling with class "result"
previous pPrevious sibling <p>
find .badgeFirst descendant with class "badge"
bodyThe whole page body
<!-- Target by id -->
<button hx-get="/data" hx-target="#content">Load</button>

<!-- Replace the button itself -->
<button hx-get="/expanded" hx-target="this" hx-swap="outerHTML">Expand</button>

<!-- Delete the containing row -->
<button hx-delete="/row/1" hx-target="closest tr" hx-swap="outerHTML">Delete</button>
Pro tip: closest is your best friend in table rows, list items, and cards — it lets each row manage itself without needing unique IDs.

Key Points

1
hx-target defaults to the triggering element if omitted
2
closest and find are extended CSS selectors unique to htmx
3
Good targeting means you rarely need unique IDs on every element
4
Pair hx-target="closest tr" with hx-swap="outerHTML" to make self-managing table rows

13. hx-boost

hx-boost="true"

Convert normal links and forms into htmx requests with a single attribute on the parent. Progressive enhancement for existing apps.

Without hx-boost

  • Every link = full page reload
  • Browser re-downloads JS/CSS
  • Flash of white between pages
  • Scroll position reset

With hx-boost

  • Links become XHR requests
  • Only <body> is swapped
  • Smooth page transitions
  • URL and history still update
<!-- Add to body — ALL links and forms get boosted -->
<body hx-boost="true">
  <nav>
    <a href="/home">Home</a>
    <a href="/about">About</a>
  </nav>
  <!-- These links now navigate without full reload -->
</body>

<!-- Opt individual links out -->
<a href="/download" hx-boost="false">Download PDF</a>
Existing apps: Add hx-boost="true" to an existing multi-page app and it instantly feels like an SPA — with zero refactoring of your routes or templates.

Key Points

1
htmx fetches the linked page and swaps only the <body> content
2
The URL updates and browser history works — back button still works
3
Falls back gracefully: if JS is disabled, links work as normal
4
Combine with hx-push-url for full navigation control

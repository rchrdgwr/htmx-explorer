5. Triggering

hx-trigger

Requests don't have to come from clicks. hx-trigger lets any DOM event — or a timer — fire a request.

TriggerWhen it firesUse case
clickMouse click (default for buttons)Any button action
changeValue changes & focus leaves (default for inputs)Select dropdowns
keyup changed delay:500ms500ms after typing stopsLive search, autocomplete
mouseenterOn hoverLazy-load tooltips / previews
loadElement enters the DOMLazy sections, skeleton loaders
revealedElement scrolls into viewportInfinite scroll
every 5sEvery 5 secondsPolling, live dashboards
submitForm submission (default for forms)Ajax forms
<!-- Live search: wait 400ms after typing stops -->
<input name="q"
   hx-get="/search"
   hx-target="#results"
   hx-trigger="keyup changed delay:400ms">

<!-- Auto-refresh every 5 seconds -->
<div hx-get="/queue-status" hx-trigger="every 5s">...</div>
Modifiers are keywords you add after the event name to change how it behaves. For example hx-trigger="keyup changed delay:400ms" means: fire on keyup — but only if the value actually changed, and only after the user has stopped typing for 400ms. Without modifiers, a search input would fire a request on every single keystroke. Common modifiers: changed (skip if value unchanged), delay:Xms (debounce), throttle:Xms (rate-limit), once (fire once and stop).

Key Points

1
Any standard DOM event works as a trigger — focus, blur, input, drop, etc.
2
changed modifier prevents duplicate requests when nothing changed
3
delay debounces — essential for search inputs to avoid hammering the server
4
Custom events work too: hx-trigger="my-event from:body"

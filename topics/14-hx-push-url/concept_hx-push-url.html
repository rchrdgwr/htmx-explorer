14. hx-push-url

hx-push-url

Browser history still works. When htmx swaps content, it can update the URL so the back button, bookmarks, and sharing all work correctly.

Click link
/search?q=python
htmx XHR
Server
Returns results HTML
swap + pushState
URL updated
Back button works
<!-- URL updates to /results when content swaps -->
<input name="q"
   hx-get="/search"
   hx-target="#results"
   hx-push-url="true">

<!-- Push a specific URL instead -->
<button
  hx-get="/items/42"
  hx-target="#detail"
  hx-push-url="/items/42">
  View Details
</button>

<!-- Replace without adding to history -->
<form hx-get="/filter" hx-replace-url="true">
Why this matters: Without URL updates, users can't bookmark a filtered search result, share a link, or use the back button. hx-push-url fixes this with no extra work.

Key Points

1
true pushes the request URL; a string pushes that specific URL
2
hx-replace-url replaces the current history entry instead of adding a new one — useful for filters
3
The server must be able to render a full page response for the pushed URL (for direct visits)
4
htmx handles the popstate event so the back button triggers a new request

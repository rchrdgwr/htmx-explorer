2. HTTP Verbs

hx-gethx-post hx-puthx-patchhx-delete

htmx works with all standard HTTP methods - making it natural to build REST-like interactions. The server stays the source of truth.

AttributeHTTP MethodTypical use
hx-getGETLoad content, search, fetch a record
hx-postPOSTCreate a record, submit a form
hx-putPUTReplace a resource entirely
hx-patchPATCHUpdate specific fields (inline edit)
hx-deleteDELETERemove a record, delete a row
<!-- Load a list -->
<button hx-get="/items" hx-target="#list">Refresh</button>

<!-- Create a new item -->
<form hx-post="/items" hx-target="#list">...</form>

<!-- Delete a row -->
<button hx-delete="/items/42" hx-target="closest tr" hx-swap="outerHTML">Delete</button>
Mental model: htmx is just making HTTP requests. Think of each button or form as a tiny fetch call - but written in HTML. The server-side routes look exactly like a normal REST API.

Key Points

1
Any element can trigger any HTTP verb - not just forms and buttons
2
The server handles business logic and returns updated HTML
3
Using the correct verb makes your API clean and RESTful for free
4
DELETE and PUT still return HTML - typically the updated/removed element

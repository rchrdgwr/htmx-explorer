16. Server-Driven UI Patterns

Where htmx really clicks. These patterns combine the concepts you've learned into complete, real-world application building blocks.

Master / Detail
Select a row → update a detail pane
hx-get="/customers/42"
hx-target="#detail-pane"
Modal Dialog
Open a modal from a server response
hx-get="/modal/edit"
hx-target="#modal-container"
CRUD Table
Create, inline-edit, delete rows
hx-post="/items" + hx-delete
hx-target="closest tr"
Multi-Step Wizard
Each step returns the next step's HTML
hx-post="/wizard/step1"
hx-target="#wizard" hx-swap="outerHTML"
Live Dashboard
OOB updates + polling
hx-trigger="every 10s"
+ hx-swap-oob on multiple widgets
Shopping Cart
Add item → update cart badge + total
hx-post="/cart/add"
+ hx-swap-oob for counter

The mental shift: Stop thinking about JSON APIs and client-side state. Think about HTML fragments and URL endpoints. The server renders; htmx delivers. That's the whole model.
About this demo: The Live Dashboard pattern's numbers come from a tiny inline <script> in the response fragment, not the server — server.py just serves static files. A real backend would return freshly computed values on each poll.

Key Points

1
Every pattern is just a combination of the 7 core concepts — nothing new to learn
2
The server stays authoritative — no client-side state synchronization problems
3
htmx applications are naturally accessible — they're still HTML pages
4
Start with hx-boost on an existing app and layer in patterns as needed

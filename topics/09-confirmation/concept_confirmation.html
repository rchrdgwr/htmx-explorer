9. Confirmation

hx-confirm

Prevent accidental destructive actions. hx-confirm shows a browser dialog before the request fires — one attribute, zero JavaScript.

Click
User clicks Delete
dialog shown
Confirm?
OK or Cancel
if OK
Request
fires normally
<!-- Simplest form -->
<button
  hx-delete="/items/42"
  hx-target="closest tr"
  hx-swap="outerHTML"
  hx-confirm="Delete this item? This cannot be undone.">
  Delete
</button>
Custom dialogs: For a nicer UX, you can intercept the htmx:confirm event and show a custom modal instead of the browser's native dialog.
Custom confirm with htmx event
// Override with a custom modal
document.addEventListener('htmx:confirm', (e) => {
  e.preventDefault();
  // true skips htmx's own confirm step on retry
  MyModal.show(e.detail.question, () => e.detail.issueRequest(true));
});

Key Points

1
If the user clicks Cancel, the request is never sent
2
Works with any verb — especially useful with hx-delete
3
The confirmation message can include dynamic text from the element
4
Intercept htmx:confirm to use any modal library instead of window.confirm()

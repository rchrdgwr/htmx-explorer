10. Inline Editing

hx-gethx-puthx-target="this"hx-swap="outerHTML"

Edit without leaving the page. Click a label — it becomes a form. Save — it becomes a label again. The server drives each state transition.

State machine
View mode
Label + Edit btn
GET /edit-form
Edit mode
Input + Save btn
PUT /save
View mode
Updated label
<!-- View state (what the server initially renders) -->
<div id="name-field">
  Ada Lovelace
  <button hx-get="/name/edit"
          hx-target="#name-field"
          hx-swap="outerHTML">Edit</button>
</div>

<!-- Edit state (what the server returns for GET /name/edit) -->
<form id="name-field" hx-put="/name"
      hx-target="this" hx-swap="outerHTML">
  <input name="name" value="Ada Lovelace">
  <button>Save</button>
</form>

<!-- Save returns the updated view state -->
Key insight: The element replaces itself with hx-target="this" + hx-swap="outerHTML". The server controls both the view and edit HTML — the client just swaps.

Key Points

1
Each state transition is just a request returning a different HTML fragment
2
The server validates and either returns the form with errors, or the saved view
3
Works at any granularity: a field, a row, a whole section
4
This is the classic "htmx moment" — most people immediately see its power here

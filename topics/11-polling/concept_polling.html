11. Polling

hx-trigger="every Xs"

Automatic refresh without WebSockets. Use hx-trigger="every Xs" to repeatedly fetch fresh content on a timer.

<!-- Polls every 5 seconds automatically -->
<div
  hx-get="/queue-status"
  hx-target="this"
  hx-trigger="every 5s">
  Loading...
</div>

<!-- Stop polling when done (server returns element without hx-trigger) -->
<div>Job complete! ✓</div>
Stop polling: When the server returns an element without the hx-trigger attribute, polling stops automatically. Use this to stop when a job completes.

WebSockets

  • Persistent connection
  • Server pushes updates
  • Complex to set up
  • Needs special infrastructure

htmx polling

  • Regular HTTP requests
  • Client pulls updates
  • One attribute to add
  • Works with any server
About this demo: server.py serves static files, so the clock response below returns the exact same fragment every time — the changing time comes from a tiny inline <script> in the fragment, not from the server. A real backend would compute fresh data on each request instead.

Key Points

1
No JavaScript event listeners or setInterval needed
2
The server controls when polling stops by omitting the trigger in the response
3
Great for job status, queue depth, live clocks, dashboards
4
For push-based updates, htmx also supports SSE and WebSockets via extensions

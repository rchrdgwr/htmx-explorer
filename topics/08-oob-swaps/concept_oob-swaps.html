8. Out-of-Band Swaps

hx-swap-oob

One server response updates multiple, unrelated parts of the page. This is one of htmx's superpowers - a single request, many updates.

One response body - two targets
Server response
<p>Item added!</p>

<div id="cart-count"
   hx-swap-oob="true">
  5 items
</div>
→
Normal swap
into hx-target
→
OOB swap
finds #cart-count anywhere
<!-- On the page -->
<span id="cart-count">0 items</span>

<!-- Inside the server's response -->
<p>Item added to cart!</p>

<!-- This part gets OOB-swapped into #cart-count -->
<span id="cart-count" hx-swap-oob="true">1 item</span>
Real-world uses: Shopping cart counter, notification badge, status bar, breadcrumb, page title - all updated from one request alongside the main content.

Key Points

1
OOB elements are matched by their id - they find themselves anywhere on the page
2
No limit on the number of OOB elements in one response
3
hx-swap-oob accepts swap strategies too: hx-swap-oob="beforeend:#list"
4
This replaces the need for global state management for many common UI patterns

7. Loading Indicators

User feedback during a request. htmx adds the htmx-request class to the triggering element while the request is in flight — use it to show spinners.

hx-indicator.htmx-indicator.htmx-request
Click
htmx-request added
request in flight
Spinner
becomes visible
response arrives
Done
htmx-request removed
<!-- htmx adds .htmx-request to the button while waiting -->
<button hx-get="/slow" hx-target="#result"
        hx-indicator="#spinner">
  Load Data
</button>

<!-- Hidden by default; .htmx-request makes it visible -->
<span id="spinner" class="htmx-indicator">Loading...</span>

/* CSS that htmx uses */
.htmx-indicator { opacity: 0; transition: opacity 200ms; }
.htmx-request .htmx-indicator { opacity: 1; }
.htmx-request.htmx-indicator { opacity: 1; }
Perceived performance: A spinner or skeleton screen makes a 1-second wait feel much shorter. Always give users feedback that something is happening.

Key Points

1
htmx adds .htmx-request to the triggering element automatically — no JS needed
2
.htmx-indicator elements are hidden by default and fade in during the request
3
Use hx-indicator="#spinner" to point to an indicator anywhere on the page
4
You can also disable the button during the request with CSS: .htmx-request { pointer-events: none; }

6. Forms

AJAX forms without AJAX code. Add hx-post to a form and htmx handles serialization, submission, and response — all in one attribute.

hx-posthx-targethx-swap

Traditional form

  • Full page reload on submit
  • Browser navigates away
  • Scroll position lost
  • Must redirect after POST
  • Hard to show inline errors

htmx form

  • No page reload
  • Response swapped inline
  • Scroll position kept
  • Server returns HTML fragment
  • Inline validation is easy
<form hx-post="/save" hx-target="#result">
  <input name="name" type="text">
  <input name="email" type="email">
  <button type="submit">Save</button>
</form>

<!-- Server returns success or validation errors as HTML -->
<div id="result"></div>
Validation pattern: On error, the server returns the form again with error messages highlighted. On success, it returns a success message or the new record. No JSON, no client-side validation library needed.

Key Points

1
htmx serializes all form fields automatically — no new FormData() needed
2
Works with file uploads too — htmx sets the right Content-Type header
3
hx-post can go on a button instead of the form to submit only specific fields
4
Use hx-indicator to show a spinner while the request is in flight

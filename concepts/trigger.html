Trigger Events

hx-trigger tells htmx when to fire the request. Default is click for buttons, change for inputs.

hx-trigger
Trigger valueWhen it firesCommon use
click On mouse click Default for buttons
change When value changes and focus leaves Default for inputs/selects
keyup changed delay:500ms 500ms after the user stops typing Live search / autocomplete
mouseenter On hover Lazy-load tooltips or previews
load When element is loaded into DOM Lazy sections, infinite scroll
every 2s Every 2 seconds Live dashboards, polling
submit Form submission Default for <form>
<!-- Fires 500ms after user stops typing -->
<input
  hx-get="/search"
  hx-target="#results"
  hx-trigger="keyup changed delay:500ms">

<!-- Polls every 2 seconds -->
<div
  hx-get="/status"
  hx-trigger="every 2s"></div>

Key Points

1
changed modifier prevents firing if the value hasn't actually changed
2
delay:Xms debounces the request — essential for search inputs
3
Multiple triggers can be combined: hx-trigger="click, keyup"
4
once modifier makes the trigger fire only the first time
